--- layout: post title: Foundation - Developing Units of Sound online categories: - Personal tags: [] status: draft type: post published: false meta: {} --- UOS online like any Rich Internet Application (RIA) will need a dedicated server, although i have thought about trying to go without (post for another day), for which there are many different directions i can go in. I have come down to the decision that standard manged hosting will not work for us as we will want, or possibly even need, different technologies on our server than most web hosts are willing to provide. Therefore my choices have come to basically Virtual Machines (VM's) or Platform as a Service (PaaS). Since our technology selection is Node.JS server installation and configuration should not bee too hard, Node is a single application that require little or no configuration, unlike Apache or Nginx, we would just need to make sure that our application was started on system start up using either system daemons or possibly some other method.

Best of both worlds.

By choosing AppFog as our PaaS we have still left the door open for VM based solution. AppFog is based on cloud foundry, an open source PaaS which will allow you to download a VM giving you your own Polyglot application server that you can then run on any VM host provider.
